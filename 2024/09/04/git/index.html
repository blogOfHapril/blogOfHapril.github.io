Git使用

发表于2024-09-04|更新于2024-09-05|Config环境搭建与使用

​ recommend: 先在远程（github etc.）建立好仓库，本地进行clone，然后再添加新文件，最后推送至远程

1.win（git bash）

​ 从永远到永远-Windows10配置多个git账号（以两个gitee账号为例）_git 账号 登录 windows-CSDN博客

.1 ssh key

​ note: 若某个项目需要使用另一个账户（非全局的），（需提前git init）在该项目文件夹下执行git config user.name "Your Name" && git config user.email "you@example.com"

​ 生成新的SSH keys（打开C:/User/用户/.ssh文件夹），bash here

github->seetings->SSH and GPG keys->SSH keys，点击New SSH key，自定义一个title（电脑型号_操作系统及版本），Key框内贴入id_rsa_jb.pub（id_rsa_bj.pub）中的内容，点击Add SSH key
多个github账号就多次执行以上两个步骤

​ 创建config文件（C:/User/用户/.ssh文件夹下，没有后缀名）

​ bash输入（测试SSH连接）：

按照提示输入yes，回车，出现**Hi mekeny1! You’ve successfully authenticated, but GitHub does not provide shell access.**就说明SSH连接正常，github上SSH keys页面的钥匙也会变成绿色；若报错：

  • git@github.com: Permission denied (publickey,password,keyboard-interactive).

  • WARNING: REMOTE HOST IDENTIFICATION HAS CHANGED!

​ 使用git别名（需提前git init，使用别名实际是为了少打点字）

​ 在要推送的项目文件夹打开git bash

​ clone项目文件到本地
