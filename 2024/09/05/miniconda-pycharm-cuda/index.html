Miniconda+Pycharm+CUDA

发表于2024-09-05|更新于2024-09-05|Config环境搭建与使用

1.miniconda+pycharm

​ 推荐python3.10版本，win终端使用conda自带的prompt，ubuntu终端在安装时选择改为base

.1 windows miniconda

​ 系统变量->path路径添加（若没有自动添加的话）

​ C:\Users\april\ .condarc添加

​ 打开Anaconda Prompt (miniconda3)，输入python -m site查看pip下载路径，打开D:\ProgramFiles\miniconda3\Lib\site.py，修改约line82、87、88处

输入python -m site再此查看是否更改

​ 命令行输入pip install numpy安装numpy，若报Defaulting to user installation because normal site-packages is not writeable，则是没有文件写入权限，修改miniconda安装文件夹的Users(APRIL\Users)的权限，pip show numpy查看numpy是否下载到目标文件夹，后pip uninstall numpy卸载即可

​ warn: 创建虚拟环境后，仍需在虚拟环境安装路径下再次进行修改USER_SITE，USER_BASE为对应虚拟环境下的\Lib\site-packages、\Scripts

.2 ubuntu miniconda

​ 下载操作系统架构对应的miniconda的sh文件

​ 修改安装路径为**/home/ha《用户名》/bin/miniconda3**；最后允许将终端改为base（即输入yes）；重开终端，出现base，并输入conda list验证安装

.3 pycharm

​ File->settings->Appearance下，Use custom font->Size(14)

​ 代码提示取消大小写匹配：settings->Editor->General->Code Completion下取消Match case

​ 取消双击shift打开全局搜索：Settings->Advanced Settings，搜索double，勾选Disable double modifier key shortcuts

2.conda

​ note: ubuntu下miniconda执行的conda、pip下载包的路径都是一样的，且二者在虚拟环境下载包的位置也都是一样的，且都位于虚拟环境的路径下

5.issues

.1 clone, install package

​ clone代码到本地时，注意官方给出的Installation中的clone命令可能带--recursive，Git 会自动初始化并更新每个子模块，这时候如果直接clone的话，子模块目录会是空的，必须带上，否则后续安装子模块的包时会报错

​ 不建议按照官方的命令单独创建环境，如conda env create -f environment.yml，而是提前建立一个以torch版本为名的环境，再按照environment.yml里的内容下载包
